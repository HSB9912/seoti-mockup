지금 여기 · Phase 5

실작동 페이지 + CMS + 보안 게이트

인증·장바구니·위시리스트·재고관리·CMS·관리자 게이트 모두 작동. 다음은 데이터 정리 → Supabase 연동.

1

탐색 · 시안 비교

완료5월 21일 ~ 22일

메인 스토어 시안 3종(소프트·미니멀·팝) → 모던 미니멀(B) 확정. 각 기능별 변형 A/B(/C)도 비교.

  • 메인 스토어 시안 3개 비교 → B 확정
  • 대량주문 A/B/C, 관리자 A/B/C, 나머지 A/B 비교
  • 변형 선택 완료: 상세 A, 결제 B, 로그인 B, 마이 B, Group C, Q&A B, About A, 공지 B, 관리자 C
2

방향 전환 · 아카이브 중심

완료5월 23일

"판매보다 작품 기록이 더 큰 목적"으로 방향성 정리. 사이트 구성 재정의.

  • 6개 메뉴 확정: Notice · Collection · Archive · Group Order · QnA · Contact
  • "대량주문" → "Group Order" 용어 통일
  • 재고 상태 3단계 단순화 (재고있음/일시품절/단종)
  • 상세페이지 배송·적립 제거 · 회원포인트 전체 제거
  • Archive·Contact 페이지 신규 작성
3

일관성 + 호스팅

완료5월 24일

페이지마다 다르던 nav를 한 파일로 통일. GitHub Pages에 자동 배포.

  • assets/nav.js 공용 네비게이션 (한 파일만 고치면 모든 페이지 반영)
  • GitHub Pages 배포: https://hsb9912.github.io/seoti-mockup/
  • 핸드폰에서 즉시 확인 가능
4

비즈니스 관리 (CMS化)

완료5월 25일

노션에서 분산 관리하던 데이터를 한 어드민으로. 자동 합산·필터·드릴다운.

  • 입점처 9곳·정산 33건·세금/경비 6건 통합 관리
  • 입점처 클릭 → 그 매장의 정산만 필터
  • 월별/입점처별 그룹 토글
  • 확인 필요 항목 자동 알림
5

실작동 + 보안 게이트

진행중지금 ⟶

단순 mockup이 아닌 실제 작동하는 페이지. localStorage 기반이라 Supabase로 갈아끼우기만 하면 됨.

  • 인증 — 로그인/가입/로그아웃 작동 (auth-b)
  • Group Order 게이트 — 비로그인 시 redirect
  • 장바구니·위시리스트 — 영구 저장 + 페이지 간 동기화
  • 주문 흐름 — 결제 → 마이페이지 주문내역
  • 관리자 게이트 — admin-guard.js (비밀번호 seoti)
  • CMS — 어드민에서 홈 텍스트·통계 편집 (admin-content.html)
  • 재고 관리 — +/-, 빠른 입고, 변동 이력, CSV (admin-inventory.html)
  • 이미지 업로드 — Archive 모달 (Cloudflare 연동 자리 마련)
🌐 사이트 ⚙️ 어드민 📦 재고 📊 비즈니스 📝 CMS

7

Supabase + Cloudflare 연동

다음 다음약 1~2주

localStorage → 실제 DB. 이미지는 Cloudflare Images로.

  • Supabase 스키마 생성 — products, orders, users, qna, vendors, settlements ...
  • RLS(Row Level Security) 활성화 — 본인 데이터만 보이게
  • 관리자 role 분리 — admin만 어드민 페이지 접근
  • assets/app.js의 API 함수 본문 교체 — fetch → supabase 호출
  • Cloudflare Images 연동 — SEOTI.img(id)가 CF URL 반환
  • 기존 데이터 마이그레이션 — 입점처·정산·이미지 일괄 업로드
8

결제 연동 · 베타 테스트

  • 외부 베타 테스터 3~5명 (모바일·PC)
  • 이상 케이스 처리 (품절 중 결제 시도, 네트워크 끊김 등)
